# 오답노트 클라우드 동기화 — 배포 가이드 (2026-04-25)

## 무엇이 바뀌었나
오답노트가 더 이상 브라우저에 저장되지 않고, **Supabase DB(클라우드)** 에 저장돼요.
즉, 다른 기기로 로그인하거나 캐시를 비워도 오답노트가 그대로 있어요.

기존 사용자(이미 오답노트가 쌓여있는 학생)는 **로그인하는 순간 자동으로** 기존 오답노트가 클라우드로 옮겨지고, 토스트 메시지가 한 번 뜹니다("이전 오답노트 N개를 클라우드로 옮겼어! ☁️").

## 배포 순서

### 1) Supabase DB 마이그레이션 (이미 실행 완료됨 ✓)
`docs/14_DB_오답노트_클라우드_동기화.sql` 을 Supabase SQL Editor 에서 실행했어요.
검증 쿼리 결과: "wrong_notes 테이블 생성 완료" / 0행 → OK.

### 2) Frontend 배포
변경된 `app.html` 을 GitHub 에 push 하면 끝이에요.
- 변경 파일: `app.html` 1개
- 별도 환경변수/시크릿 변경 없음
- 별도 SDK 추가 없음 (이미 쓰던 supabase-js 만 사용)

### 3) 검증 체크리스트 (실배포 후 한 번씩)

- [ ] **신규 사용자**: 회원가입 → 로그인 → 문제 틀리기 → 오답노트 저장 → 로그아웃 → 다시 로그인 → 오답노트 그대로 있는지 확인
- [ ] **기존 사용자(localStorage 데이터 보유자)**: 로그인 직후 토스트 "이전 오답노트 N개를 클라우드로 옮겼어!" 표시 확인. 새로고침 후에도 유지되는지 확인.
- [ ] **다중 기기**: PC 에서 푼 오답이 모바일에서 로그인했을 때 보이는지
- [ ] **캐시 비우기**: 시크릿 모드/캐시 비운 상태로 로그인해도 오답노트가 보이는지
- [ ] **백업 내보내기/불러오기**: 기존 JSON 백업 파일이 import 후에도 정상 표시되는지
- [ ] **비우기 버튼**: "오답노트 비우기" 누르면 DB에서도 진짜 삭제되는지 (다른 기기에서 로그인해도 비어있어야 함)

## 변경 요약 (참고)

### app.html 안에서 바뀐 곳
1. **WrongNote 모듈 (~9080행 근처)**
   - `_save` → no-op (DB 가 이미 영속화하므로 의미 없음)
   - `_load` → DB 에서 SELECT, 첫 호출 시 localStorage 자동 마이그레이션
   - `add` → 메모리 캐시에 push + DB INSERT (백그라운드)
   - `importBackup` → DB 에 직접 INSERT (replace 모드는 기존 행 삭제 후)
   - 새 헬퍼: `_fromDbRow`, `_toDbRow`, `_migrateFromLocalStorage`

2. **오답노트 비우기 버튼** (~9374행)
   - DB delete 호출 후 메모리 캐시 비우기

3. **세션 자동 복구 / `_studiaAfterAuth`** (~3893, 3907행)
   - `SIGNED_OUT` 시 메모리 캐시 비우기 추가 (다른 사용자 데이터 노출 방지)
   - 로그인 직후 `WrongNote._load()` 자동 호출 추가

4. **자동 호출 제거** (~9403행 → 주석 처리)
   - 페이지 로드 직후의 `WrongNote._load();` 제거
   - 이유: 로그인 전엔 user_id 가 없어 의미 없음. 로그인 직후 호출이 더 정확.

## 롤백이 필요하면
DB 변경(테이블 생성)은 그대로 둬도 안전해요. (기존 동작에 영향 없음)
프론트만 이전 커밋으로 되돌리면 localStorage 방식으로 다시 돌아가요.
단, 그 사이에 클라우드에 쌓인 오답은 사용자가 다시 풀기 전까진 화면에 안 보이는 점 유의.

## 알려진 한계 / 다음 작업
- **메모리 캐시 최대 500개**: 화면에 보이는 건 최근 500개. DB 에는 그 이상도 저장됨. 추후 페이지네이션 도입 가능.
- **add() race**: 로그인 직후 `_load` 진행 중에 사용자가 빠르게 문제를 풀어 add 가 호출되면 표시 캐시가 일시적으로 덮어써질 수 있음. 다만 DB INSERT 는 정상 진행되므로 다음 새로고침에서 정상 표시. 실사용 환경에선 거의 발생하지 않음.
