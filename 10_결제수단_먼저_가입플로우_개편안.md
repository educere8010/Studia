# 결제수단 먼저 등록 방식으로 가입 플로우 개편 — 1단계 기획서

작성일: 2026-04-24
단계: 1단계 (UI · 설계 · 약관 변경안)

---

## 1. 왜 바꾸는가

**현재 흐름 (문제점)**
1. 회원가입 → 로그인 → 앱 바로 진입
2. 7일 자동 무료체험 시작 (결제수단 없이)
3. 7일 뒤 자동으로 체험 종료 → 사용자가 "구독"을 직접 눌러야 전환
4. **대부분의 사용자는 결제 버튼을 누르지 않고 이탈** → 구독 전환율이 낮음

**새 흐름 (개선)**
1. 회원가입 → 로그인 → **결제수단 등록 화면 (필수)**
2. 결제수단 등록 완료 시점부터 7일 무료체험 시작
3. 7일 뒤 등록된 카드로 **자동 결제** (월 18,900원)
4. 사용자는 그 전 언제든 해지 가능 (프로필 메뉴에서)

이 방식을 Netflix·Disney+·토스 프라임 등 주요 구독 서비스가 모두 쓰는 **표준 freemium 패턴**이야.

## 3. 화면별 변화 (사용자 입장)

```
     ↓ 카드 등록 성공
[5] 홈 화면 (앱 본체)
     상단에 "무료체험 D-7" 배지 표시
```

### 기존 체험 중인 사용자 (그대로)
```
[1] 로그인
     ↓
[2] 홈 화면 바로 진입 (결제수단 등록 건너뜀)
     상단에 기존 "무료체험 D-N" 배지 표시
```

## 5. DB 스키마 변경안

### 추가할 컬럼
```sql
-- profiles 테이블에 추가
ALTER TABLE public.profiles
    ADD COLUMN IF NOT EXISTS billing_registered_at TIMESTAMPTZ NULL;
    -- 결제수단 등록 완료 시각. NULL이면 아직 미등록.
    -- 이 값이 세팅되는 시점에 trial_started_at / trial_ends_at도 함께 세팅.
```

### 기존 컬럼 용도 재정의
| 컬럼 | 의미 (새 흐름) |
|---|---|
| `profiles.trial_started_at` | 체험 시작일. **결제수단 등록 완료 시 세팅**. |
| `profiles.trial_ends_at` | 체험 종료일 = trial_started_at + 7일. 자동결제 예정일. |
| `subscriptions.toss_billing_key` | 토스페이먼츠 빌링키 (Phase 2에서 실제 저장) |
| `subscriptions.toss_customer_key` | 사용자별 고객키 (UUID로 생성) |
| `subscriptions.status` | 'active' / 'cancelled' / 'past_due' |

### 기존 가입자 grandfathering 로직
- 스키마 변경 실행 시점에 이미 `trial_ends_at IS NOT NULL`인 사람 = **기존 가입자**
- 마이그레이션 시 `billing_registered_at = created_at`으로 세팅 (가입 시점에 이미 체험 시작한 것으로 간주 → 결제수단 등록 화면을 다시 보지 않음)
- 신규 가입자만 `billing_registered_at IS NULL`로 시작 → 결제수단 등록 강제

### `handle_new_user()` 트리거 수정
```sql
-- 현재: trial_ends_at = NOW() + 7일 (즉시 체험 시작)
-- 변경: trial_ends_at = NULL, trial_started_at = NULL (결제 등록 시 세팅)
```

### `get_subscription_status()` 함수 수정
- 기존 반환값: `eligible, state, trial_ends_at, plan`
- 추가 state: `needs_payment_method` (billing_registered_at이 NULL인 신규 가입자)
- 클라이언트는 `state === 'needs_payment_method'`일 때 결제수단 등록 화면으로 리디렉트

### 새 RPC 함수 (Phase 2에서 작성)
```sql
-- activate_trial_with_billing(p_user_id, p_billing_key, p_customer_key)
-- 결제수단 등록 성공 시 호출:
--   profiles.billing_registered_at = NOW()
--   profiles.trial_started_at = NOW()
--   profiles.trial_ends_at = NOW() + 7일
--   subscriptions 행 INSERT (plan='trial', toss_billing_key 저장)
```
